# AI Needs Clarification Market Research Report

## 1. Executive Summary

The market does not treat early-stage clarification as a generic chatbot problem. In residential architecture, design-build, and interior-led discovery, the professional standard is a staged briefing workflow that turns vague client intent into a structured brief, explicit assumptions, and a client-alignment artifact before design generation starts.

The best Phase 4 direction is:

- use AI to progressively clarify needs,
- separate verified facts from inferred assumptions,
- generate a structured requirement brief plus a client-facing design-direction board,
- and require explicit confirmation before downstream 2D / 3D generation.

Recommended P1/P2 artifact set:

1. Structured requirement brief
2. Clarification summary with open issues and assumptions
3. Design-direction board / requirement board
4. Client confirmation step

Confidence: high for workflow shape, medium for exact artifact formatting variation by firm.

## 2. Research Objective and Decision Framing

The decision to support is:

> What should a professional AI-assisted requirement-clarification workflow look like for a residential architecture product, so the product can reduce ambiguity, lock requirements responsibly, and hand off to downstream design generation?

This research focuses on:

- residential architecture,
- townhouse / villa / apartment renovation / shophouse / home-office use cases,
- small-to-mid firm and premium client-facing workflows,
- early-stage discovery, programming, and design-direction locking.

## 3. Market Segment and Scope

Primary segment:

- residential architecture,
- renovation / remodeling,
- custom home design,
- premium design-build discovery workflows,
- adjacent interior design briefing where visual preference alignment matters.

Out of scope for recommendation:

- site verification as a fully automated AI task,
- permit-stage responsibility,
- code-compliance sign-off,
- AI replacing architect judgment.

## 5. Current Professional Workflow for Client Need Clarification

### Common stage pattern

Across the reviewed references, the early-stage workflow is usually:

1. Initial intake
2. Brief / programming discovery
3. Clarification of gaps and contradictions
4. Visual preference alignment
5. Structured summary / program lock
6. Client confirmation before concept or schematic generation

### Artifact pattern by stage

Initial intake:

- contact + project type,
- site/location basics,
- budget and timing,
- broad goals.

Programming / discovery:

- room list,
- occupancy profile,
- lifestyle and operational needs,
- must-haves / must-not-haves,
- adjacency and privacy expectations,
- work / service / parking needs.

Visual alignment:

- precedent images,
- mood / atmosphere,
- style direction,
- material and color direction,
- do / do-not examples.

Lock artifact:

- structured brief,
- assumptions list,
- requirement matrix or summary board,
- sign-off or explicit approval checkpoint.

### Where misunderstandings usually happen

Most common risk areas:

- client describes a style, but really means a mood or status signal,
- room count is given without usage logic,
- budget is stated without finish expectation,
- site data is assumed rather than verified,
- renovation constraints are under-specified,
- client thinks mood-board approval equals full scope approval.

## 6. Deliverable and Artifact Comparison

### Option A: structured brief form

Strengths:

- machine-readable,
- easy to validate,
- good for downstream systems.

Weaknesses:

- weak at emotional / aesthetic alignment by itself,
- can feel dry or incomplete to clients.

### Option B: design brief PDF / programming sheet

Strengths:

- closer to real practice,
- easier for architect and client to review together,
- better for sign-off.

Weaknesses:

- less operationally structured if used alone.

### Option C: mood board / concept board only

Strengths:

- strong for emotional alignment,
- useful in client conversation.

Weaknesses:

- too vague to lock requirements,
- dangerous if treated as full brief replacement.

### Recommended artifact set

Best-fit Phase 4 set:

1. Structured requirement brief
2. Clarification board with missing items and assumptions
3. Design-direction board
4. Confirmation checkpoint

Reason:

- this balances market credibility,
- preserves engineering structure,
- and creates a premium client-facing alignment pack.

## 7. Recommended AI-Assisted Clarification Workflow

### AI should do

- progressive questioning by section,
- contradiction detection,
- gap detection,
- summarization after each stage,
- assumption exposure,
- reformulation of vague client language into structured fields for confirmation.

### AI should infer carefully

Acceptable low-risk inference:

- normalize terms into project-type taxonomy,
- map free-text style language into a style bucket,
- infer likely follow-up questions,
- cluster requests into room/program/lifestyle/style/budget categories.

### AI should never assume without confirmation

- verified site dimensions,
- survey accuracy,
- regulatory feasibility,
- structural feasibility,
- material specification certainty,
- final room count or area ranges where user intent is still ambiguous,
- permit-readiness implications.

### Workflow recommendation

1. Intake capture
2. Section-by-section clarification
3. AI summary after each section
4. Contradiction/gap review
5. Design-direction board generation
6. Final confirmation
7. Downstream handoff

## 8. Recommended Requirement-Locking Artifact Set

The recommended Phase 4 locking set is:

### Artifact 1: structured brief

Must include:

- project type,
- project mode,
- site basics,
- room program,
- household/lifestyle needs,
- style direction,
- material/color direction,
- budget and timeline,
- must-haves / must-not-haves,
- notes and open assumptions.

### Artifact 2: clarification status board

Must include:

- what is complete,
- what is missing,
- what is inferred,
- what still needs user confirmation.

### Artifact 3: design board / requirement board

Must include:

- project overview,
- goals,
- spatial priorities,
- style references,
- material direction,
- color direction,
- atmosphere keywords,
- do / do-not direction,
- approval state.

## 9. Recommended Design Board Standard

A market-credible design board for this product should not be a decorative mood collage. It should feel like a client-ready alignment sheet.

Required sections:

- project summary,
- client goals,
- target user / household summary,
- program priorities,
- style direction,
- material and color direction,
- precedent references,
- constraints and assumptions,
- explicit do / do-not guidance,
- approval state.

Premium board qualities:

- disciplined hierarchy,
- short captions explaining why each reference matters,
- visible connection between visuals and program intent,
- clear distinction between inspiration and locked requirement.

## 10. Risks, Gaps, and Open Questions

### Risks

- AI can create false confidence if the UI hides assumptions.
- A beautiful board can be mistaken for full requirement lock.
- Too many questions upfront can damage UX.
- Too little structure keeps downstream design weak.

### Open questions for implementation

- how much file/image upload should Phase 4 include in the first release,
- whether design board references will be manual, AI-generated, or hybrid,
- whether approval is client-facing only, architect-only, or dual confirmation,
- how Phase 4 confirmation state maps into Phase 3 generation/export gates.

## 11. Conclusion

The market-valid recommendation is not “make the chatbot smarter.” The correct direction is:

- formalize clarification into sections,
- expose missing information and assumptions,
- generate a credible design-direction board,
- and require explicit confirmation before the product claims the brief is locked.

That is the strongest Phase 4 path for improving both output quality and client trust.
