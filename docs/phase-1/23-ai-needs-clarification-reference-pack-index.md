# AI Needs Clarification Reference Pack Index

## 1. Purpose of the Reference Pack

This pack collects public references that show how real architecture and design workflows:

- gather client requirements,
- structure briefs,
- align on design direction,
- and present requirement-lock artifacts professionally.

## 2. How To Use These References

Use the references for:

- workflow benchmarking,
- field model design,
- UI content strategy,
- confirmation artifact design,
- stakeholder alignment.

Do not copy them blindly. Use them to calibrate what “credible” looks like.

## 3. Client Questionnaires and Intake Forms

### Inbetween Architecture — Client Brief Questionnaire

- Link: <https://www.inbetweenarchitecture.com/wp-content/uploads/2022/03/Client-Brief-Questionnaire-1.pdf>
- Source type: architecture-firm questionnaire PDF
- Project stage: discovery / briefing
- Why it matters: shows real client-language prompts covering goals, constraints, lifestyle, and preferences
- Key takeaways: use staged questions; ask what to keep/change; capture budget and timeline early

### ID8 Architecture — Homeowner Client Questionnaire

- Link: <https://id8arc.com/wp-content/uploads/2022/04/HO_Client-Questionnaire-QA_Interactive.pdf>
- Source type: architecture-firm questionnaire PDF
- Project stage: residential intake
- Why it matters: practical homeowner-facing discovery structure
- Key takeaways: collect pain points, lifestyle needs, likes/dislikes, and scope expectations

### AIBD — Design Questionnaire for Your New Home

- Link: <https://aibd.org/document/design-questionaire-for-your-new-home/>
- Source type: professional association resource
- Project stage: early planning
- Why it matters: shows broad but credible residential discovery categories
- Key takeaways: capture family profile, room needs, budget, style, and site context together

## 4. Architectural Briefs and Programming Templates

### WBDG — Architectural Programming

- Link: <https://www.wbdg.org/design-disciplines/architectural-programming>
- Source type: professional guidance
- Project stage: pre-design / programming
- Why it matters: defines programming as a disciplined process, not casual intake
- Key takeaways: structure discovery around goals, facts, concepts, needs, and problems

### RIBA Plan of Work 2020 Overview

- Link: <https://www.architecture.com/-/media/GatherContent/Test-resources-page/Additional-Documents/BOOKLETPRINTRIBAPlanofWork2020Overviewpdf.pdf>
- Source type: professional framework PDF
- Project stage: briefing and project definition
- Why it matters: shows that “Preparation and Brief” is a formal stage with spatial requirements and project information exchange
- Key takeaways: brief quality is foundational, not optional
